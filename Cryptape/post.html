韵律

简介

Rhythm，基于「垂直韵律」构建的中文网页排版框架。

95% 的网页内容来源于文本[1]，这意味着，作为一名工程师／设计师需要花费相当多的精力在字体排印上。但现实却不是如此，绝大多数工程师／设计师仅停留在解决可读性，仅有少数人追求更好的中文阅读体验。这就是 Rhythm 制作的初衷，在保持简单易用的前提下，提供一套专为中文网页排版处理的框架。

韵律

垂直的韵律是一种设计原则，保持垂直的空间具有一致的重复、渐变、交替的比例。

《The Elements of Typographic Style》的作者 Robert Bringhurst 曾提到：「若版面要有视觉韵律感，重点在于颜色，不是五颜六色的颜色，而是黑字在白纸上所构成明暗程度的颜色」。

所谓明暗程度的颜色，实际由字号与间距所构成。若字号与间距拥有重复、渐变、交替的比例时，则形成韵律感。韵律感会让读者不自觉地寻找一种规律、可预测的模式，从而沉浸其中，感到舒适。

音阶

字号大小之间的比例，如同音乐中的音阶。作曲时以特定的音阶为基础，会形成特定的音乐风格（如：大调音阶明亮、小调音阶忧郁）。字号在排版当中亦是如此，以特定的字号比例为基础，才会形成特定的排版风格。

在 Rhythm中，字号以 15px 为基础递增或递减，主要设置如下：

body { font-size: 15px; }
h1 { font-size: 1.8rem; }
h2 { font-size: 1.6rem; }
h3 { font-size: 1.4rem; }
h4 { font-size: 1.2rem; }
h5 { font-size: 1.2rem; }
h6 { font-size: 1.2rem; }
p  { font-size: 1rem; }
small { font-size: 0.8rem; }

节拍

间距就像音乐中的时间，如同节拍。节拍是对时间的分割，倘若抢拍便失去节奏。间距则是对空间的分割，不一致的间距变化，便会失去垂直的韵律。

所谓间距，在网页排版中通过行距line-height和内／外边距padding, margin构成。

垂直的韵律关键在于基本间距，通过基本间距决定当前文本如何选择对应的行距line-height和内／外边距padding, margin，保证一致的垂直空间比例。

在 Rhythm 中，我们选择 24px 作为基本间距。从而行距line-height和内／外边距padding, margin以 24px 为倍数计算。例如：正文文本行距与下边距皆为 24px，而标题则稍有不同，行距均重置为 1 em，上下边距之和为 24px 倍数，上边距为下边距的三倍数值。

p {
line-height: 1.6em;   // 24px
margin-bottom: 1.6em; // 24px
}

h1 {
  line-height: 1em;
  margin-top: 1.33em;    // 36px
  margin-bottom: 0.44em; // 12px
}

中文排版基础

样式标准化

Rhythm 采用 sanitize.css 解决浏览器渲染元素一致性问题，并实现最佳实践（如：全局默认 a 元素没有底部装饰线，以及 nav ol, nav ul 没有相应的列表样式等等）。

Rhythm 在此基础上，对默认样式进行适当的改造，以符合中文排版的要求。

超链接

超链接 a 元素，默认样式包含 text-decoration: underline;，将其修改为自定义底部装饰线，避免中文汉字与默认下划线相接，出现 十 为 土 此类不易辨认的情况。此外添加适当的左右间距，避免相邻的超链接 a 元素底部装饰线相接。

a {
  border-bottom: 1px solid #333;
  margin: 0.125em 0;
  text-decorantion: none;

  &:hover,
  &:visited,
  &:link {
    text-decorantion: none;
  }

  &:hover {
    color: Black;
    border-bottom: 1px solid black;
  }

  &:visited {
    color: #7d7d7d;
    border-bottom: 1px dotted #9f9f9f;
  }

  &:link {
    color: #5c5c5c;
    border-bottom: 1px solid #a4a4a4;
  }
}

强调

<em> 元素与 <strong> 元素皆表示强调，但其语义有着微妙的不同。在 HTML5 草案当中，前者为 emphasized text，表示上下文中的语气加重；后者为 important text，表示在上下文中的重要性。

<strong> 元素通过加粗字重表示其重要性。而 <em> 元素则使用 text-emphasis 添加着重号样式，在浏览器不支持的情况下，可回退为伪元素方案：

em {
  position: absolute;
  top: 0.65em;
  left: 0;
  width: 100%;
  overflow: hidden;
  white-space: nowrap;
  content: "・・・・・・・・・・・・・・・・・・・・・・・・・・・・・・・・・・・・・・・・";
}

缩进

有序列表 <ol> 元素、无序列表 <ul> 元素，以及区块引用 <blockquote> 元素，在浏览器中默认 40px 缩进，将其修改为 2em（两个汉字宽度）缩进，以便上下文对齐。

一件事要做好，岂能凭你心中有一点喜欢？有一点迷恋？三天浇点水，五天上点肥？

标题

标题的字号以及上下边距在韵律已有说明，在此另外说明三点:

  1. 标题语义化
  2. 标题为宋体时的样式处理
  3. 标题连续出现时，如何收紧？

标题语义化

网页排版中的许多概念源自书籍印刷，其中标题为了区分文本内容而存在。网页中 <h1> ~ <h6> 亦是如此，我们可以将其类比为书籍中的元素，方便理解其语义，设定合适的样式：

  1. 书的名称：h1
  2. 书的每个章节标题: h2
  3. 章节内的文章标题: h3
  4. 小标题/副标题: h4, h5, h6

标题为宋体时的样式处理

当标题为宋体时（Rhythm 中的标题默认为宋体），需加强字重和增加字间距，令其与内文有所差异，突出标题效果。

标题连续出现时，如何收紧

  1. 小级标题对大级标题收紧、同级标题互相收紧；
  2. 大级标题，不对小级标题收紧；
  3. 一级标题，不对一级标题收紧；
